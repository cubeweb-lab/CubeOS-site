CubeOS — сделано с нуля

Моя собственная операционная система, написанная полностью на ассемблере NASM. Проект начинался как эксперимент, но вырос в полноценную 64-битную ОС с трёхступенчатым загрузчиком, ATA PIO драйвером, поддержкой FAT32 (в разработке) и прототипным графическим интерфейсом.

Скачать cubeos.img (v0.5-dev) GitHub

x86_64 · Long Mode

📋 Системные требования
  •   Процессор: x86_64 (Long Mode)
  •   Память: от 8 МБ (рекомендуется 64-128 МБ)
  •   Диск: 20 МБ свободного места
  •   Платформа: QEMU, VirtualBox, реальное железо (UEFI/BIOS)
🛠️ Технологии и особенности
NASM (ассемблер) 64-bit Long Mode VGA текстовый режим 80x25 GUI-прототип ATA PIO драйвер FAT32 парсер (dev)
  Трёхступенчатый загрузчик
  Shell с 14 командами

🖥️ CubeOS Shell — интерактивный терминал

Точная эмуляция командной строки CubeOS v0.6 (ATA + FAT32). Все команды работают как в реальной ОС.

cubeos@qemu: ~
==============================
CubeOS v0.6 (ATA+FAT32)
==============================
Type 'help' for available commands.
---
cubeos>
Доступно 14 команд: help, ver, about, cls, uptime, mem, cpu, dice, flip, gui, fat32, dir, reboot, shutdown

Команды reboot/shutdown в демо-режиме не перезагружают страницу. GUI — прототип, показывает оповещение.

📥 Установка и запуск

# 1. Скачать образ cubeos.img (v0.4 стабильный)
# 2. Запустить в QEMU:
qemu-system-x86_64 -drive format=raw,file=cubeos.img -m 128M

# Или на Android через Limbo x86
# Запись на флешку (осторожно!)
dd if=cubeos.img of=/dev/sdX bs=512

# Сборка из исходников (Windows):
# build.bat (требуется NASM)

Также поддерживается VirtualBox и реальное железо (Legacy BIOS). Исходный код доступен на GitHub.
